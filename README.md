# Vibe FFT

Рекурсивная и итеративная реализации быстрого преобразования Фурье (БПФ) с

* Автовекторизацией (AVX2, AVX512f) для GCC/Clang
* AVX2 для MSVC

### Требования

* **C++20**
* **Компилятор:** GCC 13+, Clang 16+ или MSVC 19.35+
* **Архитектура:** x86-64 с поддержкой AVX2 (v3)

### Сборка и запуск

Проект использует CMake и настроен на максимальную производительность (O3, AVX2, FMA):

```bash
mkdir build && cd build
cmake -DCMAKE_BUILD_TYPE=Release ..
cmake --build .
./fft_app
```

### Методология тестирования производительности

Для обеспечения воспроизводимости и научной точности замеров в проекте используются следующие техники:

*   **Thread Affinity (Привязка к ядру):** Основной поток принудительно закрепляется за логическим ядром ЦПУ (`Core 0`). Это исключает накладные расходы на переключение контекста ОС и минимизирует влияние динамического распределения нагрузки между ядрами.
*   **Warm-up (Разогрев):** Перед каждым замером выполняется серия итераций алгоритма «вхолостую». Это позволяет прогреть кэши процессора (L1/L2), заполнить таблицы предсказателя переходов и стабилизировать частоту CPU (вывод на стабильный уровень Turbo Boost / AVX Offset).
*   **Статистическая обработка данных:** 
    *   Для минимизации влияния системных прерываний в таблицу заносится **медианное значение** времени выполнения из выборки в 500–5000 итераций. 
    *   **Jitter (Вариативность):** Рассчитывается как коэффициент вариации (стандартное отклонение, деленное на среднее значение). Высокий Jitter на малых $N$ обусловлен дискретностью системного таймера; низкий Jitter на больших $N$ подтверждает стабильность работы алгоритма с подсистемой памяти.
*   **Метрики производительности:**
    *   **MFLOPS:** Рассчитывается на основе теоретической сложности БПФ: $10 \cdot N \log_2(N) + N$ операций на полный цикл (прямое + обратное преобразование + нормировка).
    *   **Cycle (ms):** Медианное время выполнения полного цикла (Forward + Inverse).
*   **Контроль точности:**
    *   **SNR (Signal-to-Noise Ratio):** Отношение энергии исходного сигнала к энергии ошибки восстановления (в дБ).
    *   **L-inf (Чебышёвская норма):** Максимальное абсолютное отклонение между исходным и восстановленным вектором. Значение `OK` выставляется при $L_{\infty} < \epsilon$, где допуск $\epsilon$ масштабируется согласно $O(\log_2 N)$ и учитывает использование агрессивных математических оптимизаций (`/fp:fast`).
